# Quick Reference: Adding a New Component

## Checklist for Adding `my-component`

### 1. Create Component Directory Structure
```
my-component/
├── src/                    # Optional: source files
├── cw-my-component.ts      # Main component file
├── styles.css              # Tailwind styles
├── package.json            # Component package config
├── tsconfig.json           # TypeScript config
├── tsconfig.build.json     # Build-specific TS config
├── vite.config.ts          # Vite build config
├── tailwind.config.js      # Tailwind config
├── postcss.config.js       # PostCSS config
├── vite-env.d.ts           # Vite type definitions
└── README.md               # Component documentation
```

### 2. Update Root `package.json`

**Add to `exports`:**
```json
"./my-component": {
  "types": "./my-component/dist/cw-my-component.d.ts",
  "import": "./my-component/dist/cw-my-component.js"
}
```

**Add to `files` array:**
```json
"my-component/dist/**/*",
"my-component/README.md"
```

**Add scripts:**
```json
"build:my-component": "cd my-component && npm run build",
"dev:my-component": "cd my-component && npm run dev",
"preview:my-component": "cd my-component && npm run preview",
"install:my-component": "cd my-component && npm install",
"clean:my-component": "rimraf my-component/dist"
```

**Update aggregate scripts:**
```json
"build": "npm run build:navigation && npm run build:my-component",
"install:all": "npm run install:navigation && npm run install:my-component",
"clean": "npm run clean:navigation && npm run clean:my-component"
```

### 3. Component File Pattern

```typescript
import { LitElement, html, css, unsafeCSS } from 'lit';
import { customElement, property } from 'lit/decorators.js';
import styles from './styles.css?inline';

@customElement('cw-my-component')
export class CwMyComponent extends LitElement {
  @property({ type: String })
  myProp: string = '';

  static styles = [
    unsafeCSS(styles),
    css`
      :host {
        display: block;
      }
    `
  ];

  render() {
    return html`
      <div class="my-component">
        ${this.myProp}
      </div>
    `;
  }
}

declare global {
  interface HTMLElementTagNameMap {
    'cw-my-component': CwMyComponent;
  }
}
```

### 4. Usage Pattern

```javascript
import '@jtimbobolan/shared-ui-components/my-component';

const component = document.querySelector('cw-my-component');
component.myProp = 'value';
```

### 5. Build and Test

```bash
# Install dependencies
npm run install:my-component

# Build
npm run build:my-component

# Development mode
npm run dev:my-component

# Test package contents
npm pack --dry-run
```

## Naming Conventions

- **Component name:** `cw-my-component` (kebab-case with `cw-` prefix)
- **Class name:** `CwMyComponent` (PascalCase)
- **File name:** `cw-my-component.ts`
- **Package name:** `@jtimbobolan/cw-my-component`

## Files to Copy from `navigation/` as Templates

1. `tsconfig.json` - TypeScript configuration
2. `tsconfig.build.json` - Build-specific TypeScript config
3. `vite.config.ts` - Vite bundler config (update component name!)
4. `tailwind.config.js` - Tailwind CSS config
5. `postcss.config.js` - PostCSS config
6. `vite-env.d.ts` - Vite type definitions
7. `styles.css` - Base Tailwind styles

Remember to update component-specific names in these files!
